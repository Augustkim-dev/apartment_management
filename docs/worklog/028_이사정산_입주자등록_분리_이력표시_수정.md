# 028. 이사정산 입주자 등록 분리 + 거주자 이력 표시 수정

**날짜**: 2026-03-05
**관련 작업**: `026_이사정산_중도퇴거_입주_분할청구`, `027_이사정산_추정청구서_뷰어`

## 개요

이사정산 기능에서 3가지 문제를 해결:

1. **입주자 등록이 정산 흐름에 불필요하게 포함됨** → 정산 폼에서 입주자 등록 단계 제거
2. **새 입주자 등록 후 과거 데이터가 새 이름/연락처로 표시됨** → tenant_name_snapshot 백필 + 쿼리 수정
3. **전화번호도 동일한 이력 문제 존재** → contact_snapshot 컬럼 추가 및 동일 패턴 적용
4. **사용자 대시보드에서 추정 고지서 접근이 불편** → 바로가기 버튼 추가

## 변경 사항

### 1. 이사정산 폼 5단계 → 4단계 축소

**수정**: `app/dashboard/move-settlements/new/page.tsx`

- Step 4 (입주자 등록) 제거 → 4단계 폼으로 변경
  - Step 1: 호실 선택
  - Step 2: 퇴거 정보 (정산일, 계량기값)
  - Step 3: 추정 미리보기
  - Step 4: 확인 및 저장
- 삭제: 입주자 관련 state 변수 6개, Step 4 UI 블록, handleSubmit의 incomingTenant 로직
- 입주자 등록은 정산 상세 페이지에서 별도로 처리

### 2. tenant_name_snapshot 백필 마이그레이션

**신규**: `scripts/migration_011_backfill_tenant_name_snapshot.sql`

- tenant_id가 있지만 snapshot이 NULL인 경우 → tenants 테이블에서 채움
- tenant_id도 NULL인 경우 → 해당 호실의 첫 번째 tenant에서 매칭

### 4. contact_snapshot 컬럼 추가 및 쿼리 수정

**신규**: `scripts/migration_012_add_contact_snapshot.sql`

- `unit_bills`에 `contact_snapshot VARCHAR(20)` 컬럼 추가
- 기존 데이터 백필 (tenant_id 기반 → 호실 첫 번째 tenant 기반 2단계)

**수정**: INSERT 시 contact 캡처 (3곳)

| # | 파일 | 함수 | 변경 |
|---|------|------|------|

| 2 | `lib/services/unit-bills.service.ts` | `saveCalculationResults` (move_in) | incoming tenant contact INSERT에 추가 |
| 3 | `lib/services/move-settlement.service.ts` | `createMoveOutSettlement` | activeTenant.contact INSERT에 추가 |

**수정**: 조회 쿼리에서 `u.contact` → `COALESCE(ub.contact_snapshot, u.contact)` (6곳)

| # | 파일 | 위치 | 변경 |
|---|------|------|------|
| 1 | `lib/services/unit-bills.service.ts` | `getMonthlyUnitBills` | `u.contact` → COALESCE |

| 3 | `lib/services/unit-bills.service.ts` | `getAllSettlements` 필터 | `u.contact LIKE` → COALESCE |

## 검증 방법

1. migration_011, 012 실행 → `SELECT COUNT(*) FROM unit_bills WHERE tenant_name_snapshot IS NULL` → 0
2. `SELECT COUNT(*) FROM unit_bills WHERE contact_snapshot IS NULL` → 0 (또는 contact 자체가 없는 경우만)
3. 이사정산 생성: 4단계 폼으로 퇴거만 처리 확인
4. 새 입주자 등록 후: 과거 청구서에 기존 입주자명/연락처, 이후 청구서에 새 입주자 정보 표시
5. 사용자 대시보드에서 "추정 고지서 보기" 버튼으로 바로 이동 확인
6. `npm run build` 성공
