# 030. 청구서 입주자 정보 데이터 소스 수정

**날짜**: 2026-03-13
**관련 작업**: `unit-bills.service.ts`, `migration_013`

## 개요

2월 청구서의 호실별 상세 내역에서 이미 퇴실한 이전 입주자의 이름과 전화번호가 표시되는 문제 수정.
원인은 청구서 생성 시 `tenants` 테이블에서 입주자 정보를 가져왔으나, 해당 테이블의 `status`가 제대로 관리되지 않아 이전 입주자 정보가 사용된 것.

## 문제

- 201호: "이항"으로 표시 → 실제 현재 입주자는 "박현주"
- `tenants` 테이블의 이전 입주자가 여전히 `status='active'`로 남아있거나, 새 입주자가 등록되지 않음
- `units` 테이블의 `tenant_name`, `contact` 컬럼이 실제 현재 입주자 정보 (정확한 데이터)

## 수정 내용

### 1. 청구서 생성 시 입주자 조회 쿼리 수정

**파일**: `lib/services/unit-bills.service.ts`

**Before:**
```sql
SELECT u.id, t.id AS tenant_id, t.name AS tenant_name, t.contact AS tenant_contact
FROM units u
LEFT JOIN tenants t ON t.unit_id = u.id AND t.status = 'active'
WHERE u.unit_number = ?
```

**After:**
```sql
SELECT u.id, u.tenant_name, u.contact AS tenant_contact,
       t.id AS tenant_id
FROM units u
LEFT JOIN tenants t ON t.unit_id = u.id AND t.status = 'active'
WHERE u.unit_number = ?
```

- `tenant_name`, `tenant_contact`를 `units` 테이블에서 가져오도록 변경
- `tenant_id`는 FK 참조용으로 `tenants` 테이블에서 유지

### 2. 2월 기존 데이터 수정 마이그레이션

**파일**: `scripts/migration_013_fix_feb_tenant_snapshots.sql`

- 2월(monthly_bill_id=22) `regular` 타입 unit_bills만 `units` 테이블 기준으로 스냅샷 업데이트
- `move_in`/`move_out` 타입은 이사정산 데이터이므로 변경하지 않음
- 과거 청구서(11월~1월)는 당시 입주자 정보 유지

## 월별 입주자 구분 방식

각 월의 `unit_bills` 레코드마다 독립적인 `tenant_name_snapshot`이 저장됨.
표시 쿼리에서 `COALESCE(ub.tenant_name_snapshot, u.tenant_name)`를 사용하여:
- snapshot이 있으면 → 해당 월 생성 당시 입주자 표시
- NULL이면 → 현재 입주자 폴백

| 월 | tenant_name_snapshot | 표시 결과 |
|---|---|---|
| 1월 | 이항 (당시 저장) | 이항 |
| 2월 (마이그레이션 후) | 박현주 | 박현주 |
| 3월~ (코드 수정 후) | 박현주 (units에서 저장) | 박현주 |

## 영향 범위

- 신규 청구서 생성: `units` 테이블 기준으로 정확한 현재 입주자 저장
- 2월 기존 청구서: 마이그레이션으로 수정
- 과거 청구서: 변경 없음 (당시 입주자 정보 유지)
