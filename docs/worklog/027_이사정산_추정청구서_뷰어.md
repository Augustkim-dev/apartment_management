# 027. 이사정산 추정 청구서 뷰어 구현

**날짜**: 2026-03-05
**관련 작업**: `026_이사정산_중도퇴거_입주_분할청구`

## 개요

이사정산 실행 후 해당 월의 monthly_bill이 아직 생성되지 않은 상태에서도 퇴거자가 추정 청구서를 확인할 수 있도록 뷰어 기능을 추가.

### 문제 상황
- 311호가 2026-03-05 퇴거하여 이사정산(id=2)을 실행
- 2월 monthly_bill이 미생성 → unit_bills 레코드가 생성되지 않음
- `createMoveOutSettlement()`는 `monthlyBillId`가 있을 때만 `unit_bills`를 INSERT하므로, 추정 요금은 계산되지만 저장되지 않음
- 관리자는 정산 관리 페이지에서 요약만 볼 수 있고, 퇴거자는 청구서를 확인할 방법이 없음

### 해결 방안
1. `move_settlements` 테이블에 항목별 추정 요금 컬럼 추가 (monthly_bill 없이도 fee 데이터 보존)
2. 추정 청구서 전용 API 생성 (fee 컬럼 NULL일 경우 estimationService로 재계산)
3. 관리자/퇴거자(로그인)/퇴거자(비로그인) 3가지 접근 경로 제공

## DB 마이그레이션

```sql
ALTER TABLE move_settlements
  ADD COLUMN estimated_basic_fee DECIMAL(10,0) DEFAULT NULL,
  ADD COLUMN estimated_power_fee DECIMAL(10,0) DEFAULT NULL,
  ADD COLUMN estimated_climate_fee DECIMAL(10,0) DEFAULT NULL,
  ADD COLUMN estimated_fuel_fee DECIMAL(10,0) DEFAULT NULL,
  ADD COLUMN estimated_power_factor_fee DECIMAL(10,0) DEFAULT NULL,
  ADD COLUMN estimated_vat DECIMAL(10,0) DEFAULT NULL,
  ADD COLUMN estimated_power_fund DECIMAL(10,0) DEFAULT NULL,
  ADD COLUMN estimated_usage_ratio DECIMAL(10,6) DEFAULT NULL;
```

## 기술 세부사항

### 추정 청구서 API 응답 구조
- `currentCharges`: 항목별 요금 (basicFee, powerFee, climateFee, fuelFee, powerFactorFee, vat, powerFund, total)
- `meterReading`: 전월지침, 당월지침, 사용량
- `usageRate`: 사용 비율
- `estimationBasis`: 참조 월, 평균 사용량/금액
- `billingPeriod`: 청구 기간
- `paymentInfo`: 납부 계좌 정보
- `isEstimated: true`: 추정 청구서 플래그

### Fallback 로직
- 기존 settlement id=2는 fee 컬럼이 NULL (마이그레이션 전 데이터)
- invoice API에서 fee 컬럼 NULL 감지 시 `estimationService`로 실시간 재계산

## 검증 방법

1. 마이그레이션 실행 → `DESCRIBE move_settlements`
2. `/api/move-settlements/2/invoice` API 응답 확인
3. `/dashboard/move-settlements/2/invoice` 관리자 뷰어
4. `/invoice/settlement/2` 공개 페이지 (비로그인)
5. 퇴거자 로그인 → `/my/move-settlements` 목록 → 상세
6. 관리자 정산 상세에서 '추정 청구서 보기' 버튼 + '공유 링크 복사' 동작 확인
